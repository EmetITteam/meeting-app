Що було у першій версії

  • Один файл index.html (~2800 рядків) — HTML + CSS + JS в одному
  • Тільки сторінка зустрічей (Meetings)
  • Базові дії: створити / редагувати / скасувати / завершити зустріч
  • Прості картки без дизайн-системи
  • Без ролей, без дашборду, без замовлень
  • Без захисту API (endpoint відкритий)
  • Без Google Calendar sync

Що додали / переробили в 4.0

Архітектура

  • Розбивка на 3 файли: index.html (shell) + css/main.css + js/app.js
  • Vercel serverless proxy замість прямих запитів з браузера в 1С
  • api/config.js — окремий endpoint для передачі токена на фронтенд
  • Sentry для моніторингу помилок на бекенді

Безпека

  • APP_SECRET_TOKEN — фронтенд отримує токен з /api/config, передає в кожний запит через X-App-Token
  • ONEC_SERVICE_TOKEN — Vercel передає токен в 1С через X-Service-Token (для відхилення прямих запитів)
  • CORS — allowedOrigins whitelist на бекенді
  • OPTIONS preflight підтримка

Дизайн — Warm Modern (Концепт 4)

  • Повний редизайн: primary #066aab, градієнт, border-radius 14–18px
  • Картки зустрічей, клієнтів, замовлень, дебіторки — єдина дизайн-система
  • FAB кнопка + з action sheet (вибір: зустріч / замовлення)
  • Адаптивна нижня навігація, відцентрована через nav-inner wrapper
  • Mobile-first, коректне відображення до 600px і ширше

Нові сторінки та модулі

  • Дашборд — виконання плану, відставання (з урахуванням робочих днів), бренди з розгортанням по клієнтах, категорії клієнтів (donut), конверсія, блок навчань (тільки для керівника) — поки моки
  • Замовлення — список, деталі, форма створення/редагування
    • Два типи документів: "Заказ покупателя" (Київ) та "Реалізація" (регіони)
    • Регіон Київ — "Реалізація" приховані
    • Статуси замовлень з кольоровими бейджами
    • Реквізит "Мероприятие" (Навчання / Проекти)
  • Клієнти — список з пошуком, картки, телефон як tel: посилання, форма додавання клієнта
  • Дебіторська заборгованість — окрема сторінка в бічному меню, картки з кольоровою смугою (прострочено / в нормі)
  • Звіти — базова структура

Валюти

  • Всі ціни в каталозі та замовленнях зберігаються в USD
  • Відображення скрізь: $X · Y грн (по курсу дня, TODAY_RATE_USD)
  • В деталях замовлення — підсумок в обох валютах

Google Calendar

  • При створенні зустрічі → автоматично створюється подія в Google Calendar менеджера
  • При оновленні → подія оновлюється
  • При відміні/завершенні → подія видаляється
  • calendarEventId зберігається в 1С через окремий updateMeetingCalendarId запит

UX-деталі

  • Телефон у картці зустрічі — tel: посилання (тап → дзвінок)
  • Телефон у картці клієнта — tel: посилання
  • Автодоповнення клієнтів у формі зустрічі та замовлення
  • Автовибір основного договору (isPrimary) при виборі клієнта
  • Маски введення номерів телефонів (IMask)

Що чекає підключення від 1С

  • Реальні дані продажів для дашборду (getSalesData, getPlanData)
  • Реальні замовлення та каталог
  • Блок навчань (регістри Мероприятий + Виды обучений)
  • Перевірка X-Service-Token на стороні HTTP-сервісу 1С
